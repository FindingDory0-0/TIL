### Context
로컬 환경에서 개발 중인 n8n 워크플로우를 24시간 안정적으로 실행하기 위해 GCP(Google Cloud Platform)의 Always Free 티어인 `e2-micro` 인스턴스를 활용한 무료 호스팅 환경을 구축하고자 함. 기존 도메인 및 SSL 설정 과정에서 발생한 기술적 난관(CAA 오류 등)을 우회하고, Ngrok을 사용하여 외부 연결을 처리하는 경량화된 배포 방식을 정리함.

### Core
`docker-compose.yml`을 사용하여 n8n을 컨테이너 기반으로 격리하고, Ngrok 터널을 통해 웹훅(Webhook) URL을 노출하는 설정임.

```yaml
version: '3'
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - NODE_ENV=production
    volumes:
      - ~/.n8n:/home/node/.n8n

  ngrok:
    image: ngrok/ngrok:latest
    restart: always
    command: http n8n:5678
    ports:
      - "4040:4040"
```

* 위 설정을 작성한 후 `docker-compose up -d` 명령어로 실행
* Ngrok 설정 파일(`ngrok.yml`)을 통해 인증 토큰(Auth Token)을 사전에 등록해야 함
* GCP 방화벽 설정에서 5678 포트 및 Ngrok 관리 포트(4040) 개방 필요

### Insight
GCP `e2-micro` 인스턴스는 무료라는 강력한 장점이 있으나, 메모리 자원(약 1GB)이 매우 제한적이므로 무거운 워크플로우 실행 시 성능 저하가 발생할 수 있음. 특히 Let's Encrypt와 같은 공인 인증서 설정 시 DuckDNS 환경에서 CAA(Certificate Authority Authorization) 레코드 제약으로 인한 발급 실패 사례가 빈번함. 이를 Ngrok으로 대체하면 복잡한 네트워크 설정 없이 외부 웹훅 수신이 가능해지나, Ngrok 무료 티어의 도메인 휘발성 이슈를 고려해야 함. 안정성을 최우선으로 한다면 추후 Cloudflare Tunnel 등을 도입하여 정적 도메인과 고정 터널링을 구축하는 방향을 권장함.
