### Context
n8n은 Node.js 기반의 워크플로우 자동화 도구로, 로컬 환경에서 소스 코드를 직접 빌드하고 수정하기 위해서는 특정 Node.js 버전 관리와 네이티브 모듈(Native Module)의 올바른 컴파일 과정이 필수적입니다. 특히 `sqlite3`나 `isolated-vm`과 같은 네이티브 의존성은 실행 환경의 Node.js 버전에 따라 재빌드가 필요하며, 이를 자동화하기 위한 환경 구성 방법을 정리합니다.

### Core

#### 1. 개발 환경 세팅 (fnm & Node.js)
`fnm`(Fast Node Manager)을 사용하여 프로젝트별로 Node.js 버전을 격리 관리합니다.

```bash
# fnm 설치 및 자동 전환 설정
brew install fnm
echo 'eval "$(fnm env --use-on-cd)"' >> ~/.zshrc
source ~/.zshrc

# Node 22 및 빌드 도구 설치
fnm install 22
npm install -g pnpm@10.22.0 node-gyp
```

#### 2. 프로젝트 초기 빌드
프로젝트 루트에 `.node-version` 파일을 생성하여 폴더 진입 시 자동으로 Node.js 22가 활성화되도록 합니다.

```bash
echo "22" > .node-version
pnpm install

# 네이티브 모듈 재빌드 (의존성 경로 확인 필요)
pushd node_modules/.pnpm/sqlite3@5.1.7/node_modules/sqlite3 && node-gyp rebuild && popd
pushd node_modules/.pnpm/isolated-vm@6.0.2/node_modules/isolated-vm && node-gyp rebuild && popd

# 전체 빌드
export PATH="./node_modules/.bin:$PATH"
pnpm build > build.log 2>&1
```

#### 3. 로컬 실행 환경 변수
로컬 개발의 편의성을 위해 보안 및 네트워크 제한을 완화한 실행 설정입니다.

```bash
N8N_SECURE_COOKIE=false \
  N8N_DEFAULT_LOCALE=ko \
  N8N_DIAGNOSTICS_ENABLED=false \
  N8N_TEMPLATES_ENABLED=false \
  NODES_EXCLUDE=[] \
  N8N_RESTRICT_FILE_ACCESS=false \
  pnpm start
```

### Insight
* **네이티브 모듈 관리**: `sqlite3`와 `isolated-vm`은 C++ 기반의 네이티브 애드온을 포함하고 있습니다. 따라서 `npm install` 후 `node-gyp`를 통한 컴파일 단계가 생략되면 실행 시 모듈 로드 에러(Module not found or Invalid architecture)가 발생할 수 있습니다. 
* **환경 변수 최적화**: `N8N_SECURE_COOKIE=false` 설정은 HTTPS 인증서 없이 로컬에서 인증 쿠키를 처리하기 위해 필수적입니다. 또한, `N8N_DIAGNOSTICS_ENABLED=false`와 `N8N_TEMPLATES_ENABLED=false`를 설정함으로써 외부 서버와의 통신을 차단하여 로컬 개발 환경의 네트워크 격리 수준을 높이고 부팅 속도를 개선할 수 있습니다.
* **프로세스 관리**: `lsof -ti :5678 | xargs kill` 명령을 통해 개발 중 포트 충돌을 즉시 해결하는 방식은 로컬 개발 워크플로우를 단순화하는 데 매우 효과적입니다.
