### Context
구글 플레이스토어에 앱을 출시한 후, 디버그(Debug) 빌드에서는 정상 작동하던 기능이 릴리즈(Release) 빌드에서 작동하지 않는 문제를 경험했습니다. 초기에는 코드 축소 및 난독화 도구인 R8/ProGuard의 설정 오류를 의심했으나, 실제 원인은 APK와 AAB(Android App Bundle) 패키징 방식의 차이에서 기인한 네이티브 라이브러리(.so) 누락이었습니다.

### Core
AAB 배포 시 네이티브 라이브러리가 포함되지 않는 문제를 해결하려면 `build.gradle.kts` 파일에 `jniLibs` 패키징 옵션을 명시적으로 설정해야 합니다.

```kotlin
android {
    // ... 기존 설정 ...

    packaging {
        jniLibs {
            // AAB 패키징 시 네이티브 라이브러리를 압축하지 않고 포함하도록 설정
            useLegacyPackaging = true
        }
    }
}
```

### Insight
*   **패키징 차이**: APK는 모든 리소스와 라이브러리를 단일 파일에 포함하지만, AAB는 설치 시점에 기기에 최적화된 리소스만을 전달합니다. 이 과정에서 네이티브 라이브러리(.so)가 기본적으로 압축되어 포함되지 않거나, 특정 환경에서 불러오지 못하는 경우가 발생할 수 있습니다.
*   **useLegacyPackaging**: 해당 옵션은 과거 APK 방식처럼 네이티브 라이브러리를 압축하지 않고 APK 파일 내에 포함하도록 강제합니다. 이는 라이브러리 로딩 이슈를 방지하는 가장 확실한 방법입니다.
*   **주의사항**: 이 옵션을 사용하면 앱의 최종 크기가 다소 증가할 수 있으므로, 프로젝트의 라이브러리 의존성을 주기적으로 검토하는 것이 좋습니다.
