# TIL: tmux + Claude Code로 멀티 에이전트 팀 구성하기 (macOS)

macOS에서 Claude Code를 사용할 때 `tmux`를 활용하면 여러 에이전트에게 역할을 분담하고, 각자의 작업을 분할 창으로 동시에 모니터링할 수 있다.

---

## 개요

Claude Code는 단일 세션에서도 강력하지만, `tmux`로 창을 분할해 **각 에이전트에게 독립된 터미널 창을 부여**하면 병렬 작업이 가능해진다. PM, 프론트엔드, 백엔드, 디자이너 등 실제 팀 구조를 터미널에 그대로 구현할 수 있다.

---

## 예시: 시스템 모니터링 프로젝트 에이전트 팀 구성

### 팀 구성

| 역할 | 이름 |
|------|------|
| PM | 홍길동 |
| 프론트엔드 개발자 | 둘리 |
| 백엔드 개발자 | 또치 |
| 디자이너 | 희동 |

### 프롬프트 예시

```
tmux로 분할 창을 만들고, 각 창에서 Claude를 실행해줘.

시스템 모니터링 프로젝트를 위한 에이전트 팀을 꾸리고 싶어:
- PM 1명: "당신은 홍길동이라는 이름의 PM입니다. 시스템 모니터링 프로젝트의 전체 일정과 요구사항을 관리하고 팀원들에게 우선순위를 할당하세요."
- 프론트엔드 개발자 1명: "당신은 둘리이라는 이름의 프론트엔드 개발자입니다. PM의 요구사항에 따라 대시보드 UI를 설계하고 반응형 웹을 개발하세요."
- 백엔드 개발자 1명: "당신은 또치이라는 이름의 백엔드 개발자입니다. 실시간 데이터 수집을 위한 API 서버와 데이터베이스 설계를 담당하세요."
- 디자이너 1명: "당신은 희동이라는 이름의 디자이너입니다. 모니터링 시스템에 최적화된 시각화 요소와 사용자 경험(UX) 가이드를 제공하세요."

각 에이전트가 자신의 창을 가지고 독립적으로 작업할 수 있도록 분할 창으로 구성해줘.
```
---

## 핵심 tmux 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl+b` + 방향키 | 창 간 이동 |
| `Ctrl+b` + `%` | 수직 분할 |
| `Ctrl+b` + `"` | 수평 분할 |
| `Ctrl+b` + `z` | 현재 창 전체화면 토글 |
| `Ctrl+b` + `d` | 세션 분리 (백그라운드 유지) |

---

## 왜 유용한가

- **병렬 작업**: 각 에이전트가 동시에 독립적인 태스크를 수행
- **실시간 모니터링**: 모든 에이전트의 진행 상황을 한 화면에서 확인
- **역할 분리**: 실제 팀 구조를 반영해 컨텍스트 오염 없이 작업 분담
- **세션 유지**: `tmux detach` 후 재접속해도 에이전트 상태 보존
