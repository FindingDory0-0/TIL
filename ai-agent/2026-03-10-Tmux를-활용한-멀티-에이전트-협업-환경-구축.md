### Context
클라우드 코드(Claude Code)를 활용하여 복잡한 프로젝트를 수행할 때, 단일 에이전트 환경의 한계를 극복하고 여러 역할을 효율적으로 분담하기 위해 터미널 멀티플렉서인 tmux를 사용하는 방법이 효과적입니다. macOS 환경에서 tmux의 창 분할(Split Pane) 기능을 활용하면, PM 및 각 직군별 에이전트를 독립된 세션에서 실시간으로 모니터링하며 협업 구조를 구축할 수 있습니다.

### Core
tmux를 사용하여 4개의 분할 화면을 생성하고 각 에이전트에게 역할을 부여하는 워크플로우 예시입니다.

* 터미널에서 새로운 tmux 세션 생성 및 창 분할:
```bash
# 세션 생성
tmux new-session -s monitoring-project

# 화면을 4분할 (Ctrl+b 후 %, 그 다음 각 영역에서 Ctrl+b 후 ")
# 각 창에서 claude 실행
claude
```

* 각 에이전트 프롬프트 설정 (각 창에서 입력):
* PM (홍길동): "당신은 홍길동이라는 이름의 PM입니다. 시스템 모니터링 프로젝트의 전체 일정과 요구사항을 관리하고 팀원들에게 우선순위를 할당하세요."
* 프론트엔드 (유남): "당신은 유남이라는 이름의 프론트엔드 개발자입니다. PM의 요구사항에 따라 대시보드 UI를 설계하고 반응형 웹을 개발하세요."
* 백엔드 (박지영): "당신은 박지영이라는 이름의 백엔드 개발자입니다. 실시간 데이터 수집을 위한 API 서버와 데이터베이스 설계를 담당하세요."
* 디자이너 (이은형): "당신은 이은형이라는 이름의 디자이너입니다. 모니터링 시스템에 최적화된 시각화 요소와 사용자 경험(UX) 가이드를 제공하세요."

### Insight
tmux를 이용한 분할 화면 환경은 각 에이전트의 사고 과정(Chain of Thought)을 동시에 시각화할 수 있어 개발자가 전체 프로젝트 흐름을 조감(Overview)하는 데 매우 유용합니다. 다만, 에이전트 간의 정보 동기화는 수동으로 이루어져야 하므로, PM 역할을 맡은 에이전트에게 팀원들의 산출물을 주기적으로 취합하고 상태를 업데이트하도록 요청하는 것이 운영 효율을 높이는 핵심 전략입니다.
