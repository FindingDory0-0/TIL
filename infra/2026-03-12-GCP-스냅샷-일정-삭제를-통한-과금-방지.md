### Context
Google Cloud Platform(GCP)에서 Compute Engine 인스턴스를 사용할 때, 비용 최적화를 위해 Always Free 티어 내에서 서비스를 운영하고자 합니다. 현재 스냅샷(Snapshot) 생성 일정(Snapshot Schedule)이 설정되어 있어 소액의 비용이 지속적으로 발생하고 있으며, 이를 제거하여 과금을 완전히 방지하고자 합니다.

### Core
스냅샷 일정(Snapshot Schedule)을 삭제하려면 먼저 해당 일정을 디스크에서 연결 해제(Detach)한 후, 일정을 삭제해야 합니다.

* gcloud CLI를 이용한 스냅샷 일정 삭제 절차:

```bash
# 1. 특정 디스크에 연결된 스냅샷 일정 확인
gcloud compute disks describe [DISK_NAME] --zone [ZONE]

# 2. 디스크에서 스냅샷 일정 제거 (연결 해제)
gcloud compute disks remove-resource-policies [DISK_NAME] \
    --resource-policies [SCHEDULE_NAME] \
    --zone [ZONE]

# 3. 스냅샷 일정 삭제
gcloud compute resource-policies delete [SCHEDULE_NAME] \
    --region [REGION]
```

* GCP 콘솔(GUI)을 이용한 방법:
* Google Cloud 콘솔에서 'Compute Engine' > '스냅샷' 메뉴로 이동합니다.
* '스냅샷 일정' 탭을 선택합니다.
* 삭제하려는 일정을 선택하고 상단의 '삭제' 버튼을 클릭합니다.
* 참고: 일정이 특정 디스크에 연결되어 있는 경우, 먼저 해당 디스크의 상세 설정으로 이동하여 '스냅샷 일정' 섹션에서 일정을 연결 해제해야 합니다.

### Insight
GCP의 Always Free 티어를 활용할 때, 스냅샷은 무료 제공 항목에 포함되지 않으므로 사소한 금액이라도 청구될 수 있습니다. 단순히 스냅샷 결과물(Snapshot)만 삭제해서는 매일/매주 반복되는 스냅샷 생성을 막을 수 없습니다. 반드시 '리소스 정책(Resource Policies)'을 삭제하여 자동화된 스냅샷 트리거를 제거해야 합니다. 인프라 운영 시 사용하지 않는 자동화 정책을 방치하면 불필요한 비용이 누적되므로, 인프라 정비 시 주기적으로 리소스 정책을 점검하는 것이 좋습니다.
