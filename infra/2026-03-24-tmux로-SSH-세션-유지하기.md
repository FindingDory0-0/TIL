### Context

* GCP 인스턴스에 SSH로 접속하여 장시간 작업을 수행해야 하는 상황이 있었다.
* SSH 세션은 네트워크 불안정, 유휴 타임아웃(idle timeout) 등의 이유로 예고 없이 끊길 수 있다.
* 세션이 끊기면 실행 중이던 프로세스가 종료되어 작업을 처음부터 다시 해야 하는 문제가 발생한다.
* 이를 해결하기 위해 터미널 멀티플렉서(Terminal Multiplexer)인 **tmux**를 활용하는 방법을 학습했다.

### Core

tmux는 서버-클라이언트 구조로 동작한다. SSH 접속 후 tmux 세션을 생성하면, 해당 세션은 **서버 프로세스**로 백그라운드에서 유지된다. SSH 연결이 끊기더라도 tmux 서버와 그 안의 프로세스는 계속 실행되며, 재접속 후 기존 세션에 다시 붙으면(attach) 작업을 이어갈 수 있다.

**세션 생성 및 기본 사용법**

```bash
# 이름을 지정하여 새 세션 생성
tmux new -s my-work

# 세션 내에서 장시간 작업 실행 (예: 빌드, 마이그레이션 등)
./long-running-task.sh
```

**세션 분리(Detach) 및 재접속(Attach)**

```bash
# 세션에서 분리 (세션은 백그라운드에서 계속 실행됨)
# 단축키: Ctrl+b → d

# SSH 재접속 후 실행 중인 세션 목록 확인
tmux ls

# 기존 세션에 다시 접속
tmux attach -t my-work
```

**SSH 접속 시 자동으로 tmux 세션에 연결하기**

`~/.ssh/config`에 아래 설정을 추가하면, SSH 접속 시 자동으로 tmux 세션을 생성하거나 기존 세션에 재접속할 수 있다.

```ssh-config
Host my-gcp-instance
    HostName <인스턴스 IP>
    User <사용자명>
    RequestTTY yes
    RemoteCommand tmux new -A -s main
```

* `tmux new -A -s main`: `-A` 플래그는 `main`이라는 세션이 이미 있으면 attach하고, 없으면 새로 생성한다.

**자주 쓰는 tmux 단축키 (Prefix: `Ctrl+b`)**

```
Ctrl+b → d    : 현재 세션에서 분리 (detach)
Ctrl+b → s    : 세션 목록 보기
Ctrl+b → $    : 현재 세션 이름 변경
Ctrl+b → c    : 새 윈도우 생성
Ctrl+b → %    : 수직 분할 (pane)
Ctrl+b → "    : 수평 분할 (pane)
```

### Insight

* tmux의 핵심 가치는 **세션 영속성(Session Persistence)**이다. SSH 연결은 클라이언트-서버 간의 네트워크 연결일 뿐이고, tmux 세션은 원격 서버의 로컬 프로세스이므로 네트워크 상태와 무관하게 유지된다.
* `nohup`이나 `screen`도 유사한 역할을 하지만, tmux는 윈도우/패널 분할, 세션 이름 관리 등 현대적인 기능을 제공하여 사실상 표준으로 자리잡았다.
* GCP뿐 아니라 AWS, 온프레미스 등 SSH 기반으로 접속하는 모든 서버 환경에서 동일하게 적용할 수 있다. 장시간 빌드, DB 마이그레이션, 로그 모니터링 등 중단되면 안 되는 작업에 특히 유용하다.
* `-A` 플래그를 SSH config와 조합하면 접속할 때마다 수동으로 tmux 명령을 입력할 필요가 없어 워크플로우가 한층 간결해진다.
