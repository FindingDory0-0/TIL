### Context
구글 클라우드 플랫폼(GCP)의 'Always Free' 티어를 활용하여 `e2-micro` 인스턴스와 30GB 표준 디스크를 운영하더라도, '스냅샷(Snapshot)'은 무료 할당량에 포함되지 않아 매달 소액(약 7~10원)의 과금이 발생할 수 있습니다. 특히 스냅샷 일정(Snapshot Schedule)이 설정되어 있으면 정기적으로 새로운 데이터 사본이 생성되어 저장소 비용이 누적되므로, 완전한 무과금 환경을 위해 해당 일정을 삭제하고 기존 데이터를 정리하는 과정이 필요합니다.

### Core
스냅샷 일정을 관리하고 삭제하기 위한 구글 클라우드 콘솔 조작 및 gcloud CLI 명령어입니다.

* **콘솔(Console) 작업 경로**
    * Compute Engine > Storage > Snapshots 메뉴 이동
    * 'Schedules' 탭을 선택하여 현재 활성화된 일정을 확인
    * 해당 일정을 선택한 후 'Delete' 버튼을 클릭하여 삭제 (단, 디스크에 연결된 경우 먼저 해제 필요)

* **gcloud CLI를 활용한 리소스 관리**
    * 디스크에서 스냅샷 일정(Resource Policy) 연결 해제:
    ```bash
    gcloud compute disks remove-resource-policies [DISK_NAME] \
        --resource-policies=[SCHEDULE_NAME] \
        --zone=[ZONE]
    ```
    * 스냅샷 일정(리소스 정책) 삭제:
    ```bash
    gcloud compute resource-policies delete [SCHEDULE_NAME] \
        --region=[REGION]
    ```
    * 기존에 생성된 개별 스냅샷 전체 삭제:
    ```bash
    gcloud compute snapshots delete [SNAPSHOT_NAME]
    ```

### Insight
GCP의 무료 등급은 30GB의 '표준 영구 디스크(Standard Persistent Disk)' 용량을 제공하지만, 이 디스크를 복사한 '스냅샷'은 별도의 저장 공간(Snapshot Storage)으로 간주되어 비용이 청구됩니다. 기술적으로 유의할 점은 '스냅샷 일정'을 삭제하더라도 **이미 생성된 기존 스냅샷들은 자동으로 삭제되지 않는다**는 것입니다. 따라서 과금을 완전히 방지하려면 일정을 삭제한 후, 스냅샷 목록에서 기존에 쌓인 데이터들을 수동으로 모두 제거해야 합니다. 또한, 특정 디스크에 스냅샷 정책이 연결되어 있는 상태에서는 정책 삭제가 불가능하므로 반드시 '연결 해제(Detach)' 단계를 선행해야 합니다.
